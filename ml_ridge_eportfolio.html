01 — The Challenge

The Problem That
Hooked Me

The brief was deceptively simple: predict California residential house prices using census-level data. Dataset had been on Kaggle forever. It felt almost too familiar. But the real challenge wasn't building a model — it was understanding why every choice mattered, and being able to defend it.

I was assigned Ridge Regression (L2 Regularization). My goal: show that coefficient shrinkage is a universal principle — similar to weight decay in neural networks — and build 15 model iterations to prove it.

02 — Honest Beginnings

How Lost I Was (V1)

I did not open the dataset and immediately know what to do. My very first run produced an R² of 0.18 and an RMSE of $119,000 — more than twice what my final model would achieve.

What went wrong in V1

Forgot StandardScaler entirely — L2 penalty hit latitude/longitude disproportionately since they're in degree units. Left ocean_proximity as a raw string column so the encoder crashed mid-fit. Did not remove outliers before the train/test split, making predictions at extreme house values wildly inaccurate.

These weren't algorithm problems — they were pipeline discipline problems. The most important lesson arrived before any hyperparameter was touched.

03 — What the Data Said

Two Insights That
Changed Everything

Before writing another line of model code I spent a full day on EDA: correlation matrices, scatter plots, and geographic distributions. Two findings reshaped my entire approach.

Feature Correlation with Target (median_house_value)
Key finding: median_income is the dominant predictor at 0.64 correlation. The four raw count features (rooms, bedrooms, population, households) are highly inter-correlated with each other (0.86–0.98) — Ridge Regression handles this multicollinearity by shrinking all of them proportionally, rather than dropping any.

The multicollinearity cluster told me I should not manually drop correlated features — Ridge was designed for exactly this situation. Manually removing them would throw away real information.

04 — Feature Engineering

Building Smarter Inputs

Raw census counts are size-biased. A neighbourhood with 10,000 total rooms tells you nothing without knowing how many households there are. I engineered three ratio features that gave the model genuine per-unit context:

Engineered Feature Definitions & Signal
ROOMS / HH
total_rooms ÷ (households + 1)
Average living space per unit
BEDS / ROOM
total_bedrooms ÷ (total_rooms + 1)
Layout density & property type
POP / HH
population ÷ (households + 1)
Neighbourhood occupancy density
All three engineered features survived Ridge's coefficient shrinkage at high alpha values — confirming they carry genuine predictive signal, not noise.
05 — Key Learning Moment

The Experiment That
Taught the Most

Midway through, I tried SelectKBest — a filter-based feature selection method — to reduce the feature set and (I assumed) improve accuracy. It backfired immediately.

SelectKBest Failure

After applying SelectKBest, R² dropped significantly. Some of the engineered ratio features were removed because their individual F-scores looked modest. Feature interactions were ignored entirely. The core lesson: Ridge Regression already handles correlated and low-signal features gracefully via coefficient shrinkage. Applying aggressive external feature selection on top made it worse, not better.

The Real Insight

Fewer features ≠ better model. Machine learning is about preserving meaningful signal while controlling noise — which is exactly what L2 regularization does far more elegantly than manual feature dropping. Let the math do the work.

06 — Model Training

15 Versions,
One Champion

I trained 15 Ridge Regression versions across a wide sweep of alpha values (0.1 → 50,000) and four solvers (auto, sag, lsqr, cholesky). Here's how the story unfolded:

V1–V4
Low Regularization — Alpha 0.1 to 10
Minimal coefficient shrinkage. Models behaved almost identically to standard linear regression. R² hovered around 0.62–0.63 with a slight overfitting gap visible on validation data.
Baseline established
V5–V7
Optimal Zone — Alpha 50 to 500
This is where regularization earned its keep. At Alpha = 500 (V6), the model found the ideal bias-variance balance — train, validation, and test R² all converged near 0.65. Generalization without sacrificing learning capacity.
★ V6 Champion — Alpha = 500
V8–V11
Aggressive Shrinkage — Alpha 1,000 to 10,000
Coefficients became too small to represent real relationships in the data. Test R² dropped steadily — to 0.61 at Alpha=1K and 0.52 at Alpha=10K. Underfitting territory. This is the weight decay analogy made concrete.
Underfitting begins
V12–V15
Solver Experiments — sag, lsqr, cholesky
Switching solvers at the optimal alpha confirmed convergence stability — results were consistent across all four solvers. Also confirmed that positive=True forced constraints destroy genuine negative relationships (like latitude), proving real-world predictors shouldn't be constrained arbitrarily.
Convergence confirmed

09 — Reflections

What This Journey
Actually Taught Me

  • Pipeline hygiene matters before any algorithm runs. Forgetting StandardScaler, mishandling missing values, or skipping outlier removal can halve your performance before a single hyperparameter is touched.
  • Feature engineering beats algorithm complexity. Three ratio features I created gave the model richer information than any raw column alone — and survived aggressive regularization.
  • Regularization is a dial, not a switch. Small alpha keeps flexibility, moderate alpha improves generalization, very large alpha destroys learning capacity. Tuning alpha is essential, not optional.
  • Fewer features is not always better. The SelectKBest experiment proved that preserving meaningful signal matters more than reducing feature count. Let regularization handle redundancy.
  • Always evaluate on train, validation, and test. Looking at only one split hides overfitting. Consistent R² across all three sets is the real signal of a model that generalises.
  • L2 and weight decay are the same idea. Ridge Regression shrinks all coefficients proportionally — neural network weight decay does the same thing to weights. Sparsity is the difference; shrinkage is universal.
